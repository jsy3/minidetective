# 토스 로그인 후속 처리 가이드

"인가 코드를 받았어요. 토큰 발급을 위해 VITE_SERVER_BASE_URL을 설정하고, 토스 로그인 API(토큰 교환)를 연동해 주세요." 문구가 나왔을 때 진행할 작업입니다.

참고: [앱인토스 개발하기 - 토스 로그인](https://developers-apps-in-toss.toss.im/login/develop.html)

---

## 1. 인가 코드 → AccessToken 흐름 요약

| 단계 | 주체 | 설명 |
|------|------|------|
| 1. 인가 코드 받기 | 앱(프론트) | `appLogin()` SDK 호출 → `authorizationCode`, `referrer` 수신 (이미 구현됨) |
| 2. AccessToken 받기 | **백엔드 서버** | 인가 코드를 토스 API로 보내 토큰 발급 (직접 구현 필요) |
| 3. 사용자 정보 등 | 앱/서버 | AccessToken으로 사용자 정보 조회 등 (선택) |

인가 코드를 AccessToken으로 바꾸는 API는 **클라이언트가 직접 호출하면 안 되고**, 반드시 **본인 백엔드 서버**를 거쳐야 합니다.

---

## 2. 개발 시 토큰 교환 연결 (프록시 방식) — 권장

**"토큰 교환 서버에 연결할 수 없어요"** 를 피하기 위해, 이 프로젝트는 **Vite 프록시**를 사용합니다.

- 앱은 **같은 출처**로 `POST /get-access-token` 호출 (예: `http://PC_IP:5173/get-access-token`).
- Vite 개발 서버가 해당 요청을 **localhost:4000** 토큰 서버로 넘깁니다.
- 따라서 **PC IP·VITE_SERVER_BASE_URL 설정 없이** 샌드박스/브라우저 모두 동작합니다.

### 할 일

1. **토큰 교환 서버 실행** (터미널 1)
   ```bash
   npm run server
   ```
   → `토큰 교환 서버: http://localhost:4000` 로그 확인.

2. **미니앱 실행** (터미널 2)
   ```bash
   npm run dev
   ```

3. **샌드박스**에서는 앱 접속 주소를 **PC IP:5173** (예: `http://192.168.151.73:5173`) 으로 열고, 로그인 시도.

이렇게 하면 휴대폰은 5173만 보며, 4000 포트는 PC 내부에서만 사용됩니다.

---

## 3. VITE_SERVER_BASE_URL (프로덕션·프록시 미사용 시)

프록시를 쓰지 않고 **직접 백엔드 URL**을 쓰는 경우에만 필요합니다.

- 프로덕션 빌드 시: 배포한 백엔드 주소를 `.env`에 설정.
- 개발 시 프록시를 끄고 테스트할 때: `VITE_SERVER_BASE_URL=http://localhost:4000` 또는 `http://<PC_IP>:4000`.

---

## 4. 백엔드 연동 (토큰 교환 API)

앱은 아래와 같이 **POST `/get-access-token`** 을 호출합니다.

- **URL**: `{VITE_SERVER_BASE_URL}/get-access-token`
- **Method**: `POST`
- **Body**: `{ "authorizationCode": "...", "referrer": "sandbox" | "DEFAULT" }`

백엔드에서는 이 요청을 받아 **토스 API**로 토큰 발급을 요청한 뒤, 결과를 그대로(또는 가공해서) 클라이언트에 넘기면 됩니다.

### 3.1 토스 API 스펙 (백엔드에서 호출)

- **URL**: `https://apps-in-toss-api.toss.im/api-partner/v1/apps-in-toss/user/oauth2/generate-token`
- **Method**: `POST`
- **Content-Type**: `application/json`
- **Body**:

| 이름 | 타입 | 필수 | 설명 |
|------|------|------|------|
| authorizationCode | string | Y | 앱에서 받은 인가 코드 |
| referrer | string | Y | `sandbox`(샌드박스) 또는 `DEFAULT`(토스앱) |

공식 문서: [2. AccessToken 받기](https://developers-apps-in-toss.toss.im/login/develop.html)

### 3.2 백엔드 구현 예시 (Node/Express)

```js
// POST /get-access-token
app.post('/get-access-token', async (req, res) => {
  const { authorizationCode, referrer } = req.body;
  if (!authorizationCode || !referrer) {
    return res.status(400).json({ error: 'authorizationCode, referrer 필요' });
  }

  const response = await fetch(
    'https://apps-in-toss-api.toss.im/api-partner/v1/apps-in-toss/user/oauth2/generate-token',
    {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ authorizationCode, referrer }),
    }
  );
  const data = await response.json();

  // 앱이 data.data.success 에서 accessToken 을 찾으므로 그대로 전달
  res.json({ data });
});
```

- 토스 응답을 `{ data }` 형태로 감싸서 보내면, 현재 앱 코드의 `data?.data?.success?.accessToken` 구조와 맞습니다.
- 실제 운영 시에는 **에러 처리**, **CORS**, **파트너 인증** 등은 앱인토스/콘솔 가이드에 맞춰 추가해야 합니다.

### 3.3 referrer 참고

- **샌드박스 앱**: `referrer` = `"sandbox"`
- **토스 앱**: `referrer` = `"DEFAULT"`  
  (앱에서 `appLogin()`으로 받은 값을 그대로 백엔드로 넘기면 됨)

## 5. 이 프로젝트에 포함된 서버 사용하기

이 저장소에는 **토큰 교환 전용 서버**가 `server/` 폴더에 포함되어 있습니다.

1. **서버 설정**  
   - `server/README.md` 참고.  
   - `server/.env.example` 을 복사해 `server/.env.server` 를 만들고, **mTLS 인증서 경로**를 설정합니다.  
   - (앱인토스 콘솔에서 발급한 `.crt`/`.key` 또는 샌드박스용 mock 인증서)
2. **서버 실행**  
   - `server` 폴더에서 `npm install` 후 `npm start`  
   - 또는 루트에서 `npm run server`
3. **프론트엔드**  
   - 루트에 `.env` 파일을 만들고 `VITE_SERVER_BASE_URL=http://localhost:4000` 설정  
   - (`.env.example` 을 `.env` 로 복사 후 필요 시 URL만 수정)
4. **미니앱 실행**  
   - `npm run dev` 재시작 후 토스 로그인 시도

## 7. "토큰 교환 서버에 연결할 수 없어요" 해결

이 프로젝트는 **Vite 프록시**를 쓰므로, 휴대폰이 4000 포트에 직접 접속할 필요가 없습니다. 아래만 확인하세요.

1. **토큰 서버 실행**  
   터미널에서 `npm run server` (또는 `cd server && npm start`).  
   `토큰 교환 서버: http://localhost:4000` 로그가 나와야 합니다.

2. **미니앱 실행**  
   다른 터미널에서 `npm run dev` 실행.  
   샌드박스에서는 앱을 **http://<PC_IP>:5173** 으로 열어야 합니다 (granite.config.ts `web.host`를 PC IP로 설정).

3. **재시작**  
   `npm run server` 또는 `npm run dev`를 변경 후 다시 실행했는지 확인하세요.

## 8. 사용자 정보 조회 및 복호화

서버에 **복호화 키**와 **AAD**를 설정해 두면, 사용자 정보 API 응답의 암호화 필드를 복호화할 수 있습니다.

- **복호화 키**: 앱인토스 콘솔에서 이메일로 수신한 Base64 키 → `server/.env.server` 의 `DECRYPTION_KEY_BASE64`
- **AAD**: 동봉된 AAD 문자열(예: `TOSS`) → `AAD_STRING`

**엔드포인트**: `GET /get-user-info`  
- **헤더**: `Authorization: Bearer <accessToken>`
- **동작**: 토스 API `login-me` 호출 후, 응답의 `name`, `phone`, `birthday`, `ci`, `gender`, `nationality`, `email` 등을 AES-256-GCM으로 복호화해 반환합니다.

프론트엔드에서 로그인 후 사용자 정보가 필요하면:

```js
import { get } from '../utils/fetcher.js';
const data = await get('/get-user-info', accessToken);
const user = data?.data?.success; // 복호화된 name, phone 등
```
